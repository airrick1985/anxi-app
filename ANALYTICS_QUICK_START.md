# 🚀 銷控統計分析 - 快速啟動指南

## 如何使用統計分析功能

### 1️⃣ 打開統計分析面板

1. 進入 **銷控系統** (Sales Control System)
2. 在頂部工具欄找到 **統計分析按鈕** 📊 (圖示為 `mdi-chart-box`)
3. 點擊按鈕打開分析面板

### 2️⃣ 選擇時間粒度

面板頂部有三個時間切換按鈕：
- **本日** 📅 - 查看今天的銷售統計
- **本週** 📆 - 查看本週(週一~週日)的統計
- **本月** 📋 - 查看本月的統計

### 3️⃣ 查看統計指標

面板分為以下幾個區域：

#### 🔹 銷售狀況 (上方卡片)
- **總戶數** - 項目中全部戶別數量
- **已售戶數** - 有銷售狀態的戶別
- **未售戶數** - 未銷售的戶別 (狀態為空)
- **總車位數** - 項目中全部車位
- **已售車位** - 已被購買的車位
- **未售車位** - 未被購買的車位

每個卡片顯示：
- 📊 數字 (主要指標)
- 📈 百分比 (占比)
- 🎨 顏色編碼 (快速識別)

#### 🔹 銷售趨勢 (中部圖表)

**左側 - 狀態分布圖 (Pie Chart)**
- 顯示各銷售狀態的戶數分布
- 點擊圖例可隱藏/顯示對應狀態
- 懸停查看詳細信息 (戶數和百分比)

**右側 - 銷售金額趨勢 (Line Chart)**
- 展示每日銷售金額變化
- X軸：日期 (MM/DD)
- Y軸：銷售金額 (萬元)
- 懸停查看該日詳細金額

#### 🔹 銷售人員排行 (下方表格)

| 欄位 | 說明 |
|------|------|
| 排名 | 🏆 第1/2/3名特別標記 |
| 銷售人員 | 銷售人員姓名 |
| 成交戶數 | 該人員成交的戶別數 |
| 銷售金額 | 該人員的總銷售金額 (房+車位+溢差) |
| 溢差價 | 該人員的溢差價合計 |

表格按銷售金額降序排列，全部顯示。

## 📊 指標說明

### 什麼是「溢差價」？

溢差價 = 成交總價 - 底價合計

- ✅ **正數 (綠色)** - 溢價銷售，高於底價
- ❌ **負數 (紅色)** - 低於底價銷售
- ⚪ **零** - 按底價銷售

### 「成交總價」包含什麼？

成交總價 = 房屋成交價 + 車位成交價 (全部關聯車位)

### 時間粒度如何定義？

- **本日** - 今天 00:00 ~ 23:59
- **本週** - 本週 週一 ~ 週日
- **本月** - 本月 1日 ~ 月末

> 注：統計基於 **簽約日期** (`payment_contract_date`)，不是小訂日期

## ⚙️ 數據更新

統計數據每次打開面板時自動計算，基於：

1. **戶別資料** (`households`)
   - 銷售狀態
   - 銷售人員
   - 成交價格
   - 簽約日期

2. **車位資料** (`parkings`)
   - 車位編號
   - 關聯戶別
   - 成交價格

3. **銷售人員** (`personnel`)
   - 人員名單

> 💡 如數據更新，點擊工具欄的「重新載入」按鈕，再打開統計面板即可查看最新數據。

## 🎨 顏色說明

| 顏色 | 含義 |
|------|------|
| 🔵 藍色 | 主要指標 (總數) |
| 🟢 綠色 | 正面指標 (已售、成功) |
| 🟠 橙色 | 中立指標 (未售) |
| 🔴 紅色 | 需注意指標 (缺陷、負數) |
| 🔷 淺藍 | 信息指標 (車位等) |

## 💡 使用技巧

### 快速比較
1. 切換時間粒度 (本日 → 本週 → 本月)
2. 對比數據變化趨勢

### 識別高績效人員
1. 查看銷售人員排行表
2. 頂部為最高績效銷售人員 (⭐ 標記)

### 監控銷售進度
1. 觀察趨勢圖的每日變化
2. 識別銷售高峰和低谷

### 了解狀態分布
1. 查看 Pie 圖中各狀態占比
2. 重點關注未售 (未銷售) 戶別比例

## ❓ 常見問題

### Q: 為什麼某些人員沒有出現在排行表中？
**A:** 只顯示至少有成交記錄的銷售人員。如果某人員只有未售或待訂戶別，不會出現在已售統計中。

### Q: 為什麼金額單位是「萬」？
**A:** 這是房地產行業的慣例。1萬 = 10,000元。

### Q: 數據多久更新一次？
**A:** 
- 自動：打開面板時
- 手動：點擊工具欄「重新載入」按鈕

### Q: 可以導出統計報告嗎？
**A:** 目前版本不支持導出，後續版本將加入此功能。可以使用瀏覽器的截圖功能。

### Q: 統計中的「成交」是根據什麼判斷的？
**A:** 根據 **銷售狀態字段** (`salesStatus_backend`) 是否有值。有任何狀態值 = 已成交，空值 = 未售。
